## 非阻塞风险

已核对 alpha strip 尺寸：

- `waving_strip_v01_alpha.png`：`1999x786`

因此它们不是 `1536x1872` 的正式 `8x9` atlas，也不能直接接入运行时。

### 2. 当前切帧只适用于 review，不适用于正式接入

自动切出的 `192x208` review 帧可以用于视觉 QA，但后续正式接入前仍要重新做：

- 统一底部锚点
- 统一缩放
- 统一每行在 atlas 中的位置

### 3. `review` 比 `waving` 更克制

这不是失败项，但后续正式生产 `review` 时可以微调：

- 保留轻度点头或轻度专注
- 不加入文字和任务监督元素
- 不把道具画大

### 4. `running-left` 不能简单镜像 `running-right`

原因：

- 当前角色有单侧耳机和单侧星形发夹，属于不对称稳定识别点。
- 直接镜像会把这些特征无依据地翻到另一侧，破坏“锁定特征”的一致性。

结论：

- `running-left` 必须单独生成并单独 QA。

## 后续六行动作计划

继续遵守：

- 一次只做一行动作
- 每行动作先生成 raw strip
- 再做透明化 alpha strip
- 再切 `192x208` review/alpha_review
- 再做人工严格 QA
- 没通过只重生当前这一行
- 9 行全部通过前，不合成正式 `spritesheet.webp`

建议顺序如下。

### 1. `jumping`（5 帧）

目标：

- 表达开心、被逗乐、共同娱乐、状态改善
- 明亮但不过度偶像化

重点约束：

- 不要速度线
- 不要舞台感
- 不要胜利 pose 夸张到像演出角色
- 仍然保持桌宠比例和单角色居中

通过门槛：

- 一眼能看出“开心跳一下”
- 不是挥手复用
- 小尺寸下腿部和身体起伏可读

### 2. `waiting`（6 帧）

目标：

- 表达陪伴等待、安静休息、安抚后的平静

重点约束：

- 不要画成睡死或瘫倒
- 不要引入床、桌椅、地毯等场景
- 保持“随时会回应你”的陪伴感

通过门槛：

- 与 `idle` 有明确区别
- 与 `failed` 不混淆
- 仍然可爱，不显得没精神到要报废

### 3. `failed`（8 帧）

目标：

- 表达低状态、轻微拒绝、过载保护

重点约束：

- 允许姿态下沉、眼神疲惫
- 不要变成恐怖、崩溃、哭花、黑化
- 不要出现夸张警告标识或任务失败 UI 感

通过门槛：

- 与 `idle`、`waiting`、`review` 明显区分
- 仍保留陪伴感和可爱度
- 能支持“动作被拒绝但角色仍有边界”的语义

### 4. `running`（6 帧）

目标：

- 表达原地活跃移动、被提起时的动态感、通用移动感

重点约束：

- 不要变成明显横向位移动作
- 不要引入速度线
- 手脚摆动要能读出“活跃中”

通过门槛：

- 与 `running-right` / `running-left` 有语义区分
- 可兼容拖拽/提起时的运动反馈

### 5. `running-right`（8 帧）

目标：

- 表达向右移动或跟随

重点约束：

- 需要明确侧向轮廓
- 保持头发、外套、耳机、发夹的同角色识别
- 不要偷成普通跑步模板

通过门槛：

- 与正面动作明显不同
- 小尺寸下仍能认出是同一个角色
- 没有因为侧面而丢失主要识别点

### 6. `running-left`（8 帧）

目标：

- 表达向左移动或跟随

重点约束：

- 单独生成，禁止直接镜像 `running-right`
- 单侧耳机和星形发夹要按同角色逻辑处理
- 保持外套和吊坠结构一致

通过门槛：

- 左右方向一眼可分
- 与 `running-right` 成对，但不是镜像伪造
- 不发生“配件反侧漂移导致像另一个角色”

## 每一行动作的固定 QA 清单

每做完一行，都按以下顺序复核：

1. 文件是否真实生成。
2. raw、alpha、review、alpha_review 是否齐全。
3. 帧数是否与计划一致。
4. 192x208 下是否可读。
5. 是否仍是同一个角色。
6. 星形发夹、单侧耳机、蓝白短外套、暖黄吊坠是否仍保留。
7. 是否出现背景、文字、网格、地面阴影、速度线、舞台偶像元素、多角色。
8. 是否误变成学习工具人、监督者或别的基础身份。
9. 是否和相邻语义动作混淆。
10. 通过后才进入下一行。
