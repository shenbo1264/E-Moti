# 角色基准图筛选与验收清单

## 一票否决

- 出现 Hatsune Miku、Vocaloid 或其他现有 IP 的可识别发型、配色、服装或轮廓。
- 出现文字、logo、水印、网格、场景、地面阴影或复杂背景。
- 角色不是同一个完整桌宠主体，或者包含多个角色。
- 基础身份被画成学习工具人、课程助手、任务监督者或桌面效率工具，而不是可爱亲近的电子宠物伴侣。
- 细节过密，缩到 `192x208` 后脸、身体和配件无法辨认。
- 配件超过 2 个核心识别点，后续动画难以稳定复现。

## 打分项

每项 1-5 分，低于 20 分不进入 shortlist。

| 项目 | 说明 | 分数 |
|---|---|---:|
| 小尺寸可读性 | 缩到 `192x208` 后轮廓、表情和主体动作空间仍清楚 |  |
| 原创性 | 与 Miku、Vocaloid、现有 Codex pet、常见商业角色无明显相似 |  |
| 动画可拆性 | 能合理做 idle、waving、review、jumping、failed、running 等动作 |  |
| 角色记忆点 | 1-2 个稳定配件或轮廓特征足以让评委记住 |  |
| 情绪价值 | 可爱、漂亮、亲近，像会回应玩家触摸和状态变化的桌面电子宠物伴侣 |  |
| 技术干净度 | 背景可移除，边缘清楚，无漂浮碎片和多余特效 |  |

## 方向专项检查

| 候选 id | 必须成立 | 不能偏向 |
|---|---|---|
| `sweet_healing_companion` | 柔软甜美、温暖补能、适合触摸/安慰/休息回应 | 过度糖果贴纸、复杂光效、学习道具身份 |
| `lively_bond_companion` | 灵动活泼、表情丰富、适合触摸/投喂/礼物/玩耍反馈 | 舞台偶像、速度线特效、复杂漂浮饰品 |
| `quiet_guardian_companion` | 安静温柔、可靠有边界、适合低状态照顾 | 计时工具、监督学习、冷淡或距离感过强 |

## 进入动画前确认

- 已选定唯一基准图。
- 已写下禁止变化项：发型、脸型、主色、服装轮廓、配件位置。
- 已确认基础身份是桌面电子宠物伴侣，学习/专注/休息只进入后续状态动作。
- 已确认左右是否对称，决定 `running-left` 后续是否允许由 `running-right` 镜像。
- 已确认不做完整 atlas，先试做 `idle`、`waving`、`review` 三行动画。
