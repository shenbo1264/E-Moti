# AI 美术 Prompt 模板

## 基准图 Prompt

```text
为一款 AI 桌面伴侣电子宠物游戏设计一个原创二次元 Q 版像素角色。
角色定位是强情绪价值的桌面电子宠物&伴侣：可爱、漂亮、亲近、会回应玩家，有被养成和陪伴在桌面的感觉；学习/专注只是她会进入的行动状态之一，不是基础身份。
角色必须是原创 OC，不是“光核”吉祥物，也不要借用任何现有 IP。
风格：可爱的 2.5 头身 chibi，清晰深色像素描边，在 192x208 小尺寸下仍然可读，透明背景。
性格：温柔、灵动、会撒娇但不过度幼稚，能通过表情和小动作回应玩家的触摸、安慰、休息、玩耍、收礼、学习等互动。
设计：原创发型，不要双马尾，不要 Hatsune Miku 轮廓，不要 Vocaloid 服装，不要复制 Miku/V 家或其他 IP 的可识别特征。
服装：简单短外套、小披肩或柔软家居感服饰，保持动画友好的清晰轮廓；加入 1-2 个稳定配件，例如耳机、发光胸针、悬浮书签或小型 AI 终端屏，不要堆叠复杂装饰。
配色：奶白、浅蓝灰、低饱和浅紫灰、暖黄色点缀，少量青蓝发光细节。
不要文字、不要 logo、不要场景、不要地面阴影、不要网格。
```

## 逐行动画 Prompt

```text
基于已经确认的原创角色设计，生成一条横向 sprite 动画 strip。
准确帧数：{frame_count}。
每一帧都必须适配一个 192x208 的透明单格。
同一个角色、同一套服装、同一个发型、同一套配色、同一比例、同一个配件。
动作：{motion_description}。
二次元 Q 版像素桌宠风格，清晰深色描边，无文字、无网格、无场景、无地面阴影。
避免 Hatsune Miku 特征：无青绿色双马尾、无 Vocaloid 服装、无复制轮廓。
```

## 动作描述

```text
idle：安静站立，轻微呼吸和眨眼。
waving：小幅友好挥手，适合轻触回应、亲近互动或收到礼物。
review：进入学习/专注状态，看着小便签或小屏幕，表现认真思考；这是具体行动状态，不是角色基础身份。
jumping：小幅开心跳跃，情绪明亮，但不越出单格。
waiting：平静坐下、陪伴等待或休息姿态，可表达专注间隙的安抚和休息。
failed：疲惫、过载、学习/专注受挫或轻微拒绝，姿态下沉但仍保持可爱陪伴感。
running-right：侧向朝右小跑。
running-left：侧向朝左小跑。
running：原地活跃移动或被提起时的动态姿态。
```
